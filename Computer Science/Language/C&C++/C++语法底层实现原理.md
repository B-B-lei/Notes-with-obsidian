1.有关内联函数的知识：￼计算机组成原理中以及机器指令相关知识我们已经了解到了许多函数调用实现的原理：涉及到位置跳转指令，而这种指令再涉及到现代CPU流水线的设计模式，导致跳转实际的开销较大；￼当一个程序内部涉及到许多的函数调用功能时，可以考虑CPP中的内联函数，用inline关键字声明；  
内联函数与普通函数调用的区别是：实际调用普通函数时是内存中近保存一份函数定义代码段，其余调用内容会将参数保存并跳转到该唯一代码段继续运行；而内联函数则是在调用的时候直接重新在该处将函数定义代码段再copy过来直接运行，不跳转了，有时这种方式更节省时间(用空间换的)。￼故当函数定义代码段的内容开销并不是很大(函数体较短)，并且涉及到程序反复多层调用该函数时，可以采用内联函数定义的方式来定义该函数，以节省时间；注意递归函数不允许内联方式定义。
 
2.引用类型变量，定义的方式与指针类型变量类似，需要加上&特殊标识符用于表示定义的数据类型是引用类型：￼int &a=b;￼需要注意到：引用型变量定义的时候必须完成初始化，这一点类似const定义的变量；另外引用型变量一旦定义为原对象变量的别名后就不可改变，后续利用该引用变量重新赋值时原对象的值也会改变；引用的实质与 type* const ptr 基本一致。

3.注：！！！￼编译器在翻译CPP源文件时，如函数，会将函数名等修饰，实现名称修饰(用于标识该部分内容)，但CPP的标准允许编译器设计者自行设计名称修饰，故存在相同源文件经过不同编译器生成的二进制模块之间无法识别名称导致链接失败的情况，就比如一个函数定义模块由A编译器编译，函数调用模块由B编译，而A与B对于同一个函数名生成的名称修饰不同，导致链接时无法识别对应函数。(当发生链接失败错误并且有源程序代码情况下，可以重新将源程序用同一个(自己的)编译器重新编译)。

6、有关头文件.h以及#include关键字的作用:￼.h头文件仅仅为一个简单的文本文件，它本身并不会像cpp文件参与实际编译过程，它起作用的地方在编译前的预处理阶段：￼#include与#define宏定义功能类似，都是将指定内容替换并插入到使用该关键字的文件当前位置；也就是说头文件这个文本文件的作用仅仅是被插入替换到使用它的源文件中，而使用实际变量与函数前一般都先声明，头文件中一般也只包含数据声明的字段，具体的实现内容放在其它.cpp文件中(需要被编译的文件)￼  
可以理解为#include .h后，相当于.h文件中文本内容插入到源文件当前位置中；而当外部文件引用全局变量前需要用extern关键字，故头文件中声明全局变量的时候都加上extern关键字修饰，这样一来引用头文件的源文件就不需要再用extern声明新变量，而是可以直接用了。
 
由于重复定义相同的内容编译器会报错，而头文件存在被不同源文件引用的情况，故写头文件的使用要使用#ifndef #define #endif 这些宏定义来实现头文件管理，保证头文件仅被引用一次。

1.CPP中异常相关知识：￼CPP中异常处理功能模块由三个关键字功能协同实现：throw,try,catch;￼￼其中throw关键字用在实际会产生异常处(比如某种condition就是异常，则使用if(condition){throw…..}) ，它的作用是抛出异常，语法功能类似return， 后面跟一个CPP内置数据类型变量或一个类对应对象(实际的值)； 但与return不同的是它并不是简单的将程序控制权按原顺序换给调用函数，而是退出该被调用函数后直接进行后续catch处理部分。
 
try{}关键字用于将程序中可疑、有风险的代码段包裹起来(比如包含判断条件与throw抛出异常语句的函数)，当{}中代码段抛出异常后，直接跳转到后续的catch语句中寻找处理逻辑；
 
catch(){}关键字用于执行捕获异常后的后续处理逻辑；()内内容与语法类似函数接收参数，()内定义对应的throw抛出的异常数据类型以及定义一个形参，该形参接收实际throw抛出的数据内容，在后续{}中处理时可直接使用该异常抛出值(基本数据类型或自定义对象)
    
2.异常处理中throw异常抛出会导致栈解退，也就是直到遇到对应throw中返回类型的异常catch捕获语句之前，会一直销毁当前栈，并清除栈中之前定义的成员变量；所以栈中若存在类也会自动调用析构函数可以实现正常的内存管理；但一旦存在自定义的new动态分配内存空间(非通过类构造方式分配，假如类内部也涉及到new动态分配)，而delete操作在throw之后，则会导致无法正确管理内存，从而内存泄露(因为动态自定义new内存是通过指针返回值接收的，而栈解退销毁栈中成员数据时只是删除了该指针，该指针指向空间并没有得到释放)；
   

3.CPP中定义新的可变指针或引用类型变量并用已有的const修饰的对应指针/引用常量值去直接赋值的语句是非法的；￼但若是想要实现修改已有的const定义数值，可通过const_cast关键字来实现const/volatile状态的切换；比如：const int* ptr1=&a;￼int* ptr2=const_cast\<int* \>(ptr1); 后续可使用ptr2来修改值了；

1.智能指针相关：￼智能指针本质为利用模板类封装的指针，指向T模板类型的指针作为类的一个成员；其与普通指针的区别在于使用指针指针的话在离开该智能指针变量作用域时编译器会自动调用该对象析构函数(而智能指针的析构函数包含delete该指针语句)，使得使用指针的用户不需要再警惕合适时使用delete回收new来的内存。
 
智能指针都有一个explicit构造函数，该关键字意味着不允许隐式转换(object=member 这种语句不合法)。而指针变量仅为智能指针对象的一个成员，故explicit限制导致不能直接使用及那个指针赋值给智能指针的语句，而是利用其构造函数如auto_ptr\<type\> pd(……)，或者用另一个智能指针变量来赋值。
   

注意不要将非new来的指针/地址赋值给智能指针，智能指针与普通指针不同，它在离开作用域回收时必定自动调用析构函数，而delete非堆空间(栈上空间)时会发生错误。
   

2.函数符(函数对象)￼为了STL中许多algorithm库中设计的方法方便使用，其中许多方法都接收函数符作为一个方法参数，并在方法内部调用该函数符；￼而函数符本质为一个对象重载了()运算符，并将对象作为方法参数传入方法，方法内部调用该对象内的成员函数(即为“()”运算符重载函数)；除此之外函数指针(函数名可被隐式转化为函数指针)也可以如此使用(作为函数符)。
