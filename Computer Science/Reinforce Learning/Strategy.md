
## Introduction
策略是RL中，衡量一个Agent“智能与否”的关键，也是RL field中train的关键内容。

我们回顾MDP(Markov Decision Process)的五元组：
(S,A,P,R,$\Upsilon$)，依次为：状态、动作、环境模型 $p(s',r|s,a)$  ，奖励、折旧率。
我们可以把 MDP 的五元组 $(S, A, P, R, \gamma)$ 看作一套**标准接口（API）**：

| **元组组件**          | **现实隐喻** | **为策略提供的服务**          |
| ----------------- | -------- | --------------------- |
| **$S$ (状态)**      | 眼睛       | 给策略提供决策的**输入依据**。     |
| **$A$ (动作)**      | 手脚       | 给策略提供决策的**输出空间**。     |
| **$P$ (环境)**      | 世界运行规律   | 决定了策略执行后的**后果随机性**。   |
| **$R$ (奖励)**      | 价值观      | 告诉策略什么是“对”，什么是“错”。    |
| **$\gamma$ (折旧)** | 远见程度     | 决定了策略是“贪婪当下”还是“布局未来”。 |

## 1.策略评估

本质就是计算Bellman Equation,参见[[Bellman Equation]] .
同时我们可以认识到：Bellman Equation本质上就是衡量当前策略的好坏的一种关键性能指标；


## 2.策略迭代

策略迭代的本质是找到比当前strategy更好的$\pi$ ，那么如何做到呢？
考虑这样一种情形：
对于当前$\pi$下固定state的价值期望：
$$V_{\pi}(s)=\sum_{a}\pi(a|s)*Q_{\pi}(s,a)\tag{2.1}$$
认识到$\pi$本身是关于给定s下采取不同a行为的概率分布模型，
很明显每个state下的V是基于当前$\pi$的期望（加权求和结果），而不同的a对应的Q一定是不同的；何不采用greedy的策略每次都选择$Q_{\pi}(s,a)的a$呢? 如此一来，我们将$\pi$更新为了：连续的概率分布模型-----> 0,...1的硬编码模型，只有max Q的a才会被$\pi$选择，进一步地，得到策略迭代公式：
$$\pi(s)=argmax_{a\epsilon\alpha}Q_{\pi}(s,a)$$
其中$\alpha$是s状态下actions的可取值空间； 这种greedy策略得到的new 策略显然能够提升每个s下的V(s),那么意味着它是比原策略更优的一个策略。
注意到这里的Q(s,a)的求解是基于：Q(s,a)与 V(s')加权的公式求解得到的； 将Bellman Equation求解得到的所有V值加权得到Q后，分析每个a下不同Q的大小，选择最大Q的那个a.

步骤：
1.根据当前策略$\pi$下的Bellman Equation计算all states' V;
2.根据每个s,a下的Q(S,A)与V(s')的关系计算每个s下，可选取a的Q;
3.更新策略，$\pi(s)$变为greedy的，只选择s下让Q(s,a)最大的那个a的策略；
4.迭代上述1->2->3的过程，直到V收敛，此时的$pi$为最优策略。
Note:
4中迭代过程中，注意到此时计算V的Bellman Equation不再是期望了，而是策略固定选择某个a的情形，但因为a会导致$p(s',r| s,a)$产生新的s'与r,所以仍可能会更新V(s)的value

Summary:
可见更新strategy 的关键在与分析Q(s,a)中让他最大的a,而Q/V都是我们“偷跑”得到的，能够衡量全局结果的value,直接比较大小就能得到当前s下的最优a; 选择greedy but work on global evalution,结果收敛后也是全局最优的（存疑，需要深度思考）。

## 一些启发式思考（intuition）:

### 1.理论上的深度递归
如果我们的situation确定，s和a都是全局固定已知的，那么每个s下的最优策略也就是全局最优解，一定是存在，已知且唯一的，我们只要把他解出来即可；

如何解决呢？最朴素的想法就是DP+递归求解:
我们从终点开始思考，每个距离最优解最近的nodes们首先更新得到他们的s下的策略，然后回退到次近的nodes们，进一步更新；

Note: 可以联想复仇者联盟里奇异博士深度递归寻找干掉灭霸的solutions.

### 2.现实中的反复单层迭代（求解线性方程组）
考虑Bellman Equation的本质（以动作价值为例）：
$$Q_{\pi}(s,a)=\sum_{s',r}p(s',r|s,a)*(r+\Upsilon*\sum_{a}\pi(a|s')*Q_{\pi}(s',a))\tag{3.2}$$
当我们需要确定s下的最佳动作a(最佳策略的一部分)时，我们需要转移后的其余状态下的最优价值$Q_{\pi}(s',a)$,同样地，想要得到该value也依赖于它的可转移状态中最优的s''状态下Q.....
这样就是一个深度遍历DFS的过程，当问题规模极其庞大（就比如棋局）时，递归操作开销将爆炸增长无法实现......
同样是Bellman Equation,我们考虑不展开$Q_{\pi}(s',a)$的解决方案：
所有不同的s下都会得到一组其余s' , s''的相关总奖励函数的线性组合（只对Bellman Equation用了一层哈），此时我们得到的是一个：线性方程组，方程组左右都是关于全局s变量的线性组合式，我们可以进行线性方程组的整合化简，这样问题就变为了：关于s变量在左侧，r(r在RL中是常量，是不会更新变化的！)在右侧作为常量，本质上变成了线性方程组的求解问题；

### 3.深度递归与广度迭代收敛到一致性

那么这样单层的求解LInear与实际需要深层次递归的求解结果有什么关系呢？
这就是策略迭代起作用的地方了：
我们最开始的策略是混乱的，那么Q,V的value也是全局低效的，但是每个s下可以选择让当前总收益最高的策略，这样每个s下的Q,V一定是在单调增长的；
我们迭代这个过程直到Q,V不再变化，那么就收敛到了全局最优；

为何全局最优？我们根据DP的思想来考虑：每个s在迭代的过程中，本质上是一个所有s协同工作的异步流程；没错这是一个异步的流程：
距离终点最近的s在最开始迭代的时候，考虑greedy策略会迅速选择对于该state下的最优解action,并且在后续其他s的迭代收敛过程中都不会再变化了... 其余node在后续的迭代轮次中会充分考虑到其他s的内容，距离已经收敛的nodes越近的节点越快收敛，这样一来收敛到全局最优actions(策略)的过程会像瘟疫一样从边界开始向最远的s节点（初始state,往往是玩家的play start）扩散开，直到所有s(nodes)全部收敛。

本质上s,s' 之间在这个过程中是异步协作的过程；
没错，这也和Bellman提出的Bellman-Ford 单源最短路径的 算法思路一致，迭代求解的过程，每个node的收敛速度依次由：距离近->远收敛，最多只需要迭代V-1(V为节点个数，最远的nodes距离源点也就是V-1条边)轮次也能保证收敛。
